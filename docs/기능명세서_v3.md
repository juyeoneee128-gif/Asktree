## **2. [이슈] 탭**

### **2.1 글로벌 헤더**

- 좌측: "Critical N" / "Warning N" / "Info N" 상태 배지
- 활성 배지: 각 레벨 컬러 배경 + 흰색 텍스트
- 비활성 배지(0건): #F5F5F4 배경, #A8A29E 텍스트
- 우측: "마지막 분석: N분 전 (세션 #N)"

### **2.2 좌측 리스트 (모든 뷰 공통)**

| 항목 | 스펙 |
| --- | --- |
| 배경 | #F5F5F4 |
| 3섹션 | "미확인 (N)" / "확인 완료 (N)" / "해결됨 (N)" |
| 섹션 헤더 | #E7E5E4 배경, padding 8px 20px, 12px Bold #78716C, chevron-down/right |
| 미확인 항목 | 이슈 레벨 배지(Critical/Warning/Info) + 제목 14px SemiBold #1C1917 + 부제 12px #A8A29E |
| 선택 항목 | #FFFFFF 배경 + 좌측 이슈 레벨 컬러 3px 세로선 |
| 확인 완료 항목 | 제목 #78716C + 부제 #A8A29E + 체크 아이콘 #1C1917 |
| 해결됨 항목 | 제목 #A8A29E + 취소선 + 부제 #D6D3D1 + 체크 아이콘 #D6D3D1 |
| 구분선 | #E7E5E4 1px |

### **2.3 우측 상세 패널**

**순서 (위→아래):** Fact → Detail → Fix → 기술 근거 → 안내 문구 → 확인 완료 버튼

| 요소 | 스펙 |
| --- | --- |
| 타이틀 | 이슈 레벨 배지 + 이슈명 24px Bold #1C1917 + 파일명 14px #78716C |
| Fact 카드 | #FFFFFF 배경, 좌측 이슈 레벨 컬러 3px border-left (Critical=#DC2626, Warning=#F97316) |
| Detail 카드 | #FFFFFF 배경, 기술 상세 설명 |
| Fix 박스 | --primary 투명도 5% 배경, "복구 명령어 — Claude Code에 붙여넣으세요" 레이블 (--primary), 모노스페이스 폰트, "복사" 버튼 우측 하단 |
| 기술 근거 카드 | #FFFFFF 배경, 3열 고정: "관련 파일 / 감지 근거 / 감지 시간". 레이블 12px #A8A29E, 값 14px SemiBold #1C1917 |
| 안내 문구 | ⓘ 14px #A8A29E + "확인 시 이 기능의 보호 규칙(CLAUDE.md) 추가를 제안합니다." 12px #A8A29E |
| 확인 완료 버튼 | 패널 맨 하단. #44403C 배경, 흰색 텍스트, border-radius 12px, 전체 너비 |

### **2.4 이슈 뷰 목록 (6개)**

| 뷰 | 좌측 리스트 | 우측 패널 |
| --- | --- | --- |
| Critical | 미확인 4건 펼침 + 확인완료/해결됨 0건 접힘 | Critical 이슈 상세 |
| Warning | 미확인 4건 펼침 + 확인완료/해결됨 0건 접힘 | Warning 이슈 상세 |
| 일부 해결 | 미확인 1건 + 확인완료 2건 + 해결됨 1건 | Warning 이슈 상세 |
| 재감지 | 미확인 1건(재감지 배지) + 확인완료 1건 + 해결됨 1건 | 재감지 이슈 상세 |
| 모든 이슈 해결 | 미확인/확인완료 0건 접힘 + 해결됨 4건 펼침 | 안내 메시지 (체크 아이콘 + "현재 감지된 문제가 없습니다") |
| 빈 상태 | 미확인/확인완료/해결됨 0건 접힘 | 안내 메시지 (search 아이콘 + "아직 감지된 이슈가 없습니다" + "분석 실행" 버튼 240px) |

## **3. [현황] 탭**

### **3.1 글로벌 헤더**

- 좌측: "기획 대비 구현 현황" + 프로그레스 바 + 퍼센트
- 우측: "구현 N" "부분 N" "미구현 N" (+ "확인 필요 N" 있을 때만) + "마지막 분석" + "분석 실행" 버튼

### **3.2 좌측 기능 리스트**

| 항목 | 스펙 |
| --- | --- |
| 배경 | #F5F5F4 |
| 기능 항목 | 상태 dot(컬러) + 기능명 14px #1C1917 |
| 상태 dot | 구현=네이비, 부분=주황, 미구현=빈 원, 확인필요=⚠ 아이콘 |
| 선택 항목 | #FFFFFF 배경 + 좌측 상태 컬러 3px 세로선 |

### **3.3 우측 상세 패널**

| 요소 | 스펙 |
| --- | --- |
| 타이틀 | 상태 아이콘 + 기능명 + " — " + 상태 텍스트(컬러) + ⓘ info 아이콘 |
| 메트릭 카드 | 3열: 구현 항목(N/N), 감지된 이슈(N건), 최근 변경(세션 #N) |
| 구현 항목 리스트 | ✓ 체크 + 항목명 + " · LN" 라인 넘버 인라인 |
| 기획서 참고 카드 | FRD/PRD 원문 요약 + "원문 보기 →" 링크 |
| 기술 상세 카드 | 3열: 사용 기술, 관련 파일, 관련 세션 |
| 확인 필요 알림 | 빨간 테두리 카드 + "→ 이슈에서 Fix 확인" 링크 |

### **3.4 타이틀 상태 컬러 + 툴팁**

| 미구현 | #A8A29E (회색) | ☐ | "코딩을 시작하면 자동으로 감지됩니다" |

## **7. 설정 (프로젝트 설정)**

### **7.1 레이아웃**

사이드바 + 메인 콘텐츠. 메인에 3열 카드 그리드 + 하단 위험 영역 분리.

### **7.2 카드 3개**

| 카드 | 내용 |
| --- | --- |
| 프로젝트 정보 | 프로젝트명(수정 가능) + 생성일 + 사용 도구. "프로젝트명 수정 →" 링크 → 모달 |

## **15. 보안 정책 (MVP)**

| 항목 | 스펙 |
| --- | --- |
| API 키 저장 | Supabase pgcrypto 확장, AES-256 서버 사이드 암호화. 복호화 키는 환경변수 관리. |
| 로컬→서버 통신 | HTTPS 필수(TLS 1.2+). 인증 토큰(JWT) 헤더 포함. |
| 데이터 전송 범위 | JSONL 로그 + 파일 구조(이름/경로/크기, 내용 아님) + 변경 diff. 코드 원본은 조건부 전송(Ephemeral Processing). |
| Ephemeral Processing | 코드 원본 전송 시, 분석 완료 즉시 파기. TTL + 강제 삭제 cron 적용. 분석 결과(메타데이터)만 DB 저장. |
| 인증 | Google OAuth (최소 스코프: email, profile). 세션 토큰은 httpOnly cookie 저장. |
