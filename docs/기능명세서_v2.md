## **4. [이슈] 탭**

### **4.1 공통 레이아웃**

| 항목 | 내용 |
| --- | --- |
| 구조 | 사이드바 + 글로벌 헤더 + 좌측 이슈 리스트(35%) + 우측 상세 패널(65%) |
| 글로벌 헤더 | #F1F5F9 배경. 이슈 레벨 배지: "Critical N" (빨강 pill) / "Warning N" (노랑 pill) / "Info N" (파랑 pill). 0건이면 비강조(회색). + "마지막 분석: N분 전" |
| 좌측 리스트 | 이슈 레벨 태그(pill) + 이슈 제목 + 파일명/세션 정보. 선택 항목: --card 배경 + 좌측 레벨 컬러 3px 선 |
| 우측 패널 | --gray-50 배경. Fact → Detail → Fix 카드 구조 |

### **4.2 이슈 상세 — Fact → Detail → Fix 구조**

| 카드 | 내용 |
| --- | --- |
| 타이틀 | 레벨 태그 + 이슈 제목 (22px, Bold) + 세션/파일 정보 |
| Fact 카드 | --card 배경. 비즈니스 언어로 "뭐가 문제인지" 한 줄 설명 |
| Detail 카드 | --card 배경. Critical: 세션 비교 2열 테이블 (Before/After + 라인 범위). Warning: 텍스트 설명 (세션 비교 아님, 정적 분석) |
| Fix 카드 | --primary 투명도 5% 배경 + --primary 테두리. 레이블: "복구 명령어 — Claude Code에 붙여넣으세요". 자연어 CLI 명령어 + "복사" 버튼 (박스 안) |
| 확인 완료 버튼 | Outline 스타일, 전체 너비. 아래: "확인 시 이 기능의 보호 규칙(CLAUDE.md) 추가를 제안합니다." |
| 기술 근거 카드 | 가로 3열: 관련 파일 / 감지 세션(또는 감지 기준) / 감지 시간 |

### **4.3 Critical 이슈**

| 항목 | 내용 |
| --- | --- |
| Detail | 세션 비교 2열 테이블. 좌측: 이전 세션(줄 수) + 존재했던 함수/기능 (✓ + 라인 범위). 우측: 현재 세션(줄 수) + "삭제됨" (--color-error) |
| Fix 예시 | "dashboard.tsx에서 삭제된 사용자 권한 확인, 데이터 불러오기, 에러 처리 기능을 모두 복구해줘..." |

### **4.4 Warning 이슈**

| 항목 | 내용 |
| --- | --- |
| Detail | 텍스트 설명 (세션 비교 아님). 코드 자체의 문제 설명 |
| 감지 기준 | OWASP Top 10 등 기술 근거 명시 |
| Fix 예시 | "config.ts에서 Supabase API 키를 환경변수(.env 파일)로 분리해줘..." |

### **4.5 일부 해결 뷰**

| 항목 | 내용 |
| --- | --- |
| 좌측 리스트 | 3개 섹션: "미확인 (N)" / "확인 완료 (N)" + 체크 아이콘 / "해결됨 (N)" + 취소선, --muted-foreground |
| 해결됨 섹션 | 접기/펼치기 가능 |

### **4.6 재감지 뷰**

| 항목 | 내용 |
| --- | --- |

## **12. 상태 전이 규칙**

### **12.1 이슈 생명주기**

```
미확인 → [유저가 "확인 완료" 클릭] → 확인 완료 → [다음 분석에서 해결 확인] → 해결됨
미확인 → [유저가 "확인 완료" 클릭] → 확인 완료 → [다음 분석에서 미해결] → 재감지 (미확인으로 복귀 + "재감지" 배지)
```

### **12.2 [현황] 상태 판정**

```
기획서 기능 vs 코드 대조:
- 모든 하위 항목 구현됨 → "구현 완료"
- 일부 하위 항목 구현됨 → "부분 구현"
- 하위 항목 0개 구현됨 → "미구현"
- 구현됨이지만 기획서와 불일치 감지 → "확인 필요"
```

### **12.3 CLAUDE.md 가이드라인 상태**

```
[이슈]에서 "확인 완료" → CLAUDE.md 탭에 "미적용"으로 추가
유저가 "CLAUDE.md에 복사" 클릭 → "적용 완료"로 상태 변경
```

### **12.4 글로벌 헤더 배지 규칙**

```
[현황] 헤더: "구현 N" / "부분 N" / "미구현 N" 항상 표시. "확인 필요 N"은 1건 이상일 때만 표시.
[이슈] 헤더: "Critical N" / "Warning N" / "Info N" 항상 표시. 0건이면 비강조(회색).
사이드바 "이슈" 배지: 미확인 이슈 건수. 0건이면 배지 미표시.
사이드바 "CLAUDE.md" 배지: 미적용 가이드라인 건수. 0건이면 배지 미표시.
```
