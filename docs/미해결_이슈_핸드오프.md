## 1. 크레딧 수치 미확정 (PRD 8절)

**상태:** 추가 분석 후 확정 예정 (타래 결정)

| 항목 | 현재 상태 | 필요 작업 |
|------|----------|----------|
| 신규 가입 무료 크레딧 | "___점" 미확정 | 온보딩 1회 분석 + 기획서 1회 + 추가 2~3회 커버 가능한 수치 산정 |
| 분석 1회당 크레딧 소모량 | "약 N크레딧" 미확정 | Claude API 토큰 비용 기반 산정 필요 (50파일 프로젝트 기준) |
| 기획서 분석 크레딧 소모량 | "약 N크레딧" 미확정 | 동일 |
| 자동 분석 일일 횟수 상한 | 미확정 | 크레딧 급소진 방지용. 예: 하루 최대 5회 |
| 월정액 가격 + 포함 크레딧 | 미확정 | 비즈니스 모델 분석 |

**권장:** Claude API 비용 추정을 먼저 수행 (세션당 10만~36만 토큰, $0.30~$2.00 예상) 후 크레딧 환산 비율 확정.

## 3. 보안 설계 미완 (구현 전 해결 필수)

### Critical (즉시)

| # | 이슈 | 필요 작업 |
|---|------|----------|
| C-1 | **curl \| sh 설치 = RCE 벡터** | 체크섬/서명 검증 도입. 또는 npm/brew 설치 옵션 제공. |
| C-2 | **에이전트→서버 인증 메커니즘 부재** | 에이전트 등록 토큰 + API 키 + HMAC 서명 프로토콜 설계 |
| C-3 | **JSONL 내 민감 데이터 전송** | JSONL 내 민감 패턴 마스킹 규칙 정의 (.env 값, API 키 정규식) |
| C-4 | **API 키 저장 암호화 방식 미정의** | AES-256-GCM + KMS 분리 저장 설계 |

### High (1주 이내)

| # | 이슈 | 필요 작업 |
|---|------|----------|
| H-1 | Ephemeral Processing 기술 구현 명세 | 메모리 처리 + 디스크 미저장 + Vercel 로그 제외 설정 상세화 |
| H-2 | 파일 감시 범위 제한 | chokidar 감시 경로 제한 + 심볼릭 링크 비추적 + 제외 패턴 |
| H-3 | Supabase RLS 정책 | 모든 테이블 RLS 활성화 + 사용자별 접근 정책 설계 |
| H-4 | 기획서 업로드 보안 | 파일 크기 제한 + 타입 검증 + XXE 방지 + 샌드박스 파싱 |
| H-5 | 세션 토큰 관리 | httpOnly + Secure + SameSite 쿠키 정책 확정 |
| H-6 | Claude API 코드 전송 고지 | 이용약관/프라이버시 정책에 제3자 데이터 공유 명시 |

### Medium (2주 이내)

| # | 이슈 | 필요 작업 |
|---|------|----------|
| M-1 | JSONL 프롬프트 인젝션 방지 | AI 프롬프트에서 사용자 데이터 격리 구조 |
| M-2 | XSS 방지 (Activity 원문 렌더링) | DOMPurify 또는 텍스트 전용 렌더링 |
| M-3 | CORS/보안 헤더 | Next.js 보안 헤더 설정 |
| M-4 | 에이전트 자동 업데이트 코드 서명 | 업데이트 시 서명 검증 매커니즘 |
| M-5 | GDPR 대응 | 데이터 내보내기/삭제 기능, 프라이버시 정책 초안 |

## 4. 여전히 남은 제품/기술 리스크

### Critical 리스크 (구조적 문제)

| 리스크 | 상세 | 대응 방향 |
|--------|------|----------|
| **JSONL 포맷 비공식 의존** | Anthropic이 Claude Code JSONL 포맷을 변경/삭제하면 즉시 서비스 중단 | Anthropic에 공식 로그 API 존재 여부 확인. 에이전트에 포맷 버전 감지 + 자동 알림 메커니즘 구축 |
| **로컬 에이전트 설치 장벽** | 비개발자에게 curl + Node.js 요구. Windows 미지원. | npm/brew 설치 옵션 또는 웹 기반 GitHub 연동 Phase 2 앞당기기 검토 |

### High 리스크

| 리스크 | 상세 | 대응 방향 |
|--------|------|----------|
| Fix 실패 → 2차 피해 | 비개발자가 Fix 실패 시 판단 불가. 재감지 루프 = AI death loop | Fix 실패 2회 이상 시 "git 이전 버전 복구 권장" 대안 제시. 무한 Fix 방지. |
| CLAUDE.md 보호 규칙 과적화 | 30개+ 규칙이 쌓이면 Claude Code가 아무것도 못하는 악순환 | 규칙 개수 상한 (예: 20개) 또는 자동 정리/병합 메커니즘 |
| Alert Fatigue | 50파일에서 Warning 20~30건 + Info 10~20건 한꺼번에 표시 | 레벨별 기본 접기/펼치기 (Info는 기본 접힘). 이슈 그룹핑. |
| 자동 분석 크레딧 급소진 | 빈번한 세션 종료로 크레딧 급소진 | 일일 자동 분석 횟수 상한 + 최소 세션 간격 쿨다운 |
| Claude Code 플랫폼 의존 | Anthropic이 감지 기능 내장하면 존재 이유 소멸 | 멀티 플랫폼 지원 Phase 2 일정 앞당기기 검토 |
| 가격 카니발리제이션 | API 키 연동 유저 = Asktree 수익 0 | 프리미엄 기능 차별화 또는 API 키 유저에게도 인프라 비용 과금 |

## 5. PRD ↔ 기능명세서 잔여 불일치

v9.0에서 해소하지 못한 항목:

| # | 불일치 | 설명 | 대응 |
|---|--------|------|------|
| 1 | [현황] "확인 필요" 뷰 UX 동선 | PRD: 현황 안에서 Fact/Detail/Fix 직접 표시. 기능명세서: "이슈에서 Fix 확인" 링크만. | 하나로 확정 필요. 권장: 기능명세서 방식 유지 (이슈 탭으로 링크) |
| 2 | "확인 완료" → CLAUDE.md 추가 트리거 | PRD: "제안". 기능명세서 12.3: "자동으로 미적용에 추가". | 권장: 확인 완료 시 자동 추가 + "보호 규칙이 추가되었습니다" 토스트 안내 |
| 3 | 프로젝트 삭제 경로 통일 | 설정 탭: "되돌릴 수 없습니다" (영구 삭제). 메인 홈: 카드 더보기에 삭제 + 사이드바에 휴지통 존재. | 삭제 동작 통일 필요. 권장: 모든 삭제 → 휴지통(30일 보관 후 영구 삭제) |
| 4 | Reverse IA 편집/병합 규칙 | PRD에 Reverse IA 명시. 추출 후 유저 편집, 나중에 기획서 추가 시 병합 로직 미정의. | Phase 2에서 수동 매칭 추가 시 함께 정의 |
| 5 | 기획서 파일 크기/개수 제한 | 양쪽 모두 미정의. | 가드레일: 파일 10MB, 최대 10개 첨부 권장. 기능명세서에 추가 필요. |

## 6. 다음 세션 권장 작업 순서

```
1. [필수] 누락 화면 사양 작성 (회원가입, 관리자 설정, 플랜/결제, 크레딧 충전)
2. [필수] 보안 C-1~C-4 설계 (에이전트 인증 프로토콜, API 키 암호화)
3. [필수] 크레딧 수치 확정 (Claude API 비용 추정 → 크레딧 환산)
4. [High] 잔여 PRD↔기능명세서 불일치 5건 확정
5. [High] 보안 H-1~H-6 설계
6. [Medium] 보안 M-1~M-5 설계
7. [Medium] 가드레일 수치 확정 (파일 크기, 분석 빈도, 규칙 개수 상한)
```

## 7. 이번 세션 변경 요약 (v8.0 → v9.0)

### 확정된 결정 6건

| # | 결정 | 반영 위치 |
|---|------|----------|
| 1 | Ephemeral = 소스코드 원문만 파기. 구조 메타데이터 + JSONL 보존. | PRD 5.1 |
| 2 | 크레딧 = 포인트가 아닌 "크레딧" 명칭으로 통일. 차감 방식. 수치는 추가 분석 후 확정. | PRD 8, 기능명세서 전체 |
| 3 | 전송 데이터 = JSONL 로그 + 메타데이터. 프로젝트 전체 파일 전송 아님. | PRD 4.5 |
| 4 | 자동 분석 + 크레딧 자동 소모. 분석 전 토스트 알림으로 크레딧 안내. | PRD 4.6, 기능명세서 10.3 |
| 5 | 의도적 변경 구분 메커니즘 추가. Critical 이슈에 "의도 확인" 단계. | PRD 4.1, 기능명세서 4.2, 12.1 |
| 6 | GitHub 연동 MVP 제외. Phase 2 확정. 온보딩에서 제거. | PRD 11, 기능명세서 1.2 |
