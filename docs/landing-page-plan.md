# Asktree 랜딩 페이지 기획안 v2

> 이 문서를 Claude Code에 전달하여 랜딩 페이지 개발을 지시하세요.
> 서비스명이 "Revis"로 변경될 수 있으나, 현재는 "Asktree"로 진행합니다.

---

## 제품 정보

- **서비스명**: Asktree (변경 가능)
- **태그라인**: "바이브코더를 위한 코드 가디언"
- **한 줄 정의**: AI가 코딩하는 동안 코드를 자동으로 감시하고, 문제를 먼저 알려주고, 고치는 법을 제공하고, 다시는 같은 문제가 안 생기게 보호합니다.
- **타겟**: 비개발자 바이브코더 (Claude Code 사용자)
- **레퍼런스**: SonarQube의 비개발자 버전. Manyfast 랜딩 구조 참고.

---

## 개발 위치

- `app/(landing)/page.tsx` — 메인 랜딩 (인증 안 된 유저가 `/`에 접속 시)
- `app/(landing)/layout.tsx` — 랜딩 전용 레이아웃 (사이드바 없음)
- 인증된 유저는 `/projects`로 리다이렉트 (기존 미들웨어 유지)

---

```
[Asktree 로고]   주요 기능 ▾   가격 정책   FAQ   피드백          [로그인]
```

- **주요 기능** (드롭다운, 하위 페이지는 Phase 2에서 구현):

  - 현황 트래킹
  - CLAUDE.md 보호
  - 세션 로그
- **가격 정책**: 같은 페이지 내 앵커 스크롤
- **FAQ**: 같은 페이지 내 앵커 스크롤
- **피드백**: 외부 링크 (Google Form 또는 Tally)
- **로그인**: `/auth/login`으로 이동

## 섹션 구성 (7단계)

### 섹션 1: 히어로

**목적**: 3초 안에 "이게 뭔지" 파악 + CTA 클릭 유도

- **서브 텍스트**: "바이브코더를 위한 코드 가디언"
- **헤드라인**: **"코딩하는 동안, Asktree가 계속 지켜봅니다"**
- **서브 헤드라인**: "AI가 코드를 망가뜨려도 알 수 없었던 문제, 이제 매 세션마다 자동으로 찾아내고, 고치는 법을 알려주고, 다시는 안 망가지게 지켜줍니다."
- **CTA 버튼**: "무료로 시작하기" (10크레딧 무료, 카드 등록 없이)
- **히어로 이미지**: 실제 이슈 탭 스크린샷 (Manyfast처럼 제품 화면 삽입)
  - 스크린샷은 `/public/landing/` 에 저장, placeholder 이미지로 대체 가능

### 섹션 2: 문제 제기 (공감)

**목적**: 타겟 유저의 고통을 정확히 짚어 "나도 그래!" 반응 유도

- **헤딩**: "혹시 이런 경험 있으신가요?"
- **3가지 카드** (아이콘 + 텍스트):
  1. "AI한테 '리팩토링해줘' 했더니 3일간 만든 기능이 통째로 사라진 적"
  2. "빌드는 되는데, 결제가 안 되거나 로그인이 깨진 걸 하루 뒤에 발견한 적"
  3. "뭐가 잘못됐는지 모르겠어서 프리랜서 개발자를 불러야 했던 적"

### 섹션 3: 가치 제안 (3열 카드)

**목적**: Asktree가 뭘 해주는지 큼직하게 전달 (기능이 아닌 가치 중심)

**3열 그리드** (Manyfast 스타일, 아이콘 + 제목 + 설명):

1. **"보이지 않는 위험을 먼저 알려줍니다"** (이슈 감지)
   - "AI가 기존 기능을 삭제해도, 빌드는 통과합니다. 에러가 안 뜨니까 알 수도 없죠. Asktree는 매 세션마다 코드를 비교해서, 빌드는 되지만 기능이 망가진 문제를 자동으로 찾아냅니다."
   
2. **"기획서 대비 어디까지 왔는지 보여줍니다"** (현황 트래킹)
   - "기획서에 적은 7개 기능 중 3개가 아직 미완성입니다. 결제 연동부터 해결하세요.' — 코드를 몰라도 프로젝트의 현재 상태를 한눈에 파악할 수 있습니다."
   
3. **"한 번 당한 문제는 두 번 없게 지켜줍니다"** (CLAUDE.md 보호)
   - "감지된 문제를 CLAUDE.md 보호 규칙으로 자동 생성합니다. 규칙이 쌓일수록 AI가 함부로 건드리지 못하는 선순환이 만들어집니다."

### 섹션 4: 작동 방식 (How it works)

**목적**: "어떻게 쓰는 거지?"를 4단계로 시각화

**4단계 수평 스텝** (번호 + 아이콘 + 제목 + 설명):

1. **에이전트 설치** — "터미널에 한 줄 붙여넣기. 원클릭 설치."
2. **평소처럼 코딩** — "Claude Code로 작업하세요. 백그라운드에서 자동 수집."
3. **자동 분석** — "세션이 끝나면 Asktree가 코드를 분석합니다."
4. **이슈 확인 + Fix** — "문제를 확인하고, Fix를 복사해서 붙여넣기. 보호 규칙 추가."

### 섹션 5: 혜택 (기능이 아닌 결과)

**목적**: 기능 나열이 아니라 "그래서 나한테 뭐가 좋은데?" 답변

**3열 카드** (아이콘 + 제목 + 설명):

1. **"코딩하면 자동으로 분석"**
   - "따로 뭔가를 실행하거나 설정할 필요 없습니다. 에이전트가 백그라운드에서 알아서 합니다."

2. **"코드를 몰라도 괜찮습니다"**
   - "모든 안내는 비개발자 언어로 제공됩니다. '이 함수를 복원하세요'가 아니라 '결제 기능이 삭제되었습니다. 이 명령어를 붙여넣으세요'라고 알려줍니다."

3. **"내 코드는 서버에 저장되지 않습니다"**
   - "분석 후 즉시 파기됩니다(Ephemeral Processing). 분석 결과만 저장합니다."

### 섹션 6: FAQ

**목적**: 구매 망설임(반론)을 사전 해소

- **Q. 코드를 모르는데 쓸 수 있나요?**
  → "네. Asktree의 모든 안내는 비개발자 언어로 제공됩니다. Fix 명령어를 복사해서 Claude Code에 붙여넣기만 하면 됩니다."

- **Q. 내 코드가 서버에 저장되나요?**
  → "아닙니다. 코드 원본은 분석 후 즉시 파기됩니다(Ephemeral Processing). 분석 결과만 안전하게 저장됩니다."

- **Q. Claude Code 외에 다른 도구도 지원하나요?**
  → "현재는 Claude Code 전용입니다. Cursor, Windsurf 등은 곧 지원 예정입니다."

- **Q. 무료로 쓸 수 있나요?**
  → "가입 시 10크레딧이 무료로 제공됩니다. 본인의 Claude API 키를 등록하면 추가 비용 없이 계속 사용할 수 있습니다."

- **Q. SonarQube와 뭐가 다른가요?**
  → "SonarQube는 개발자용입니다. Asktree는 같은 가치(이슈 감지 + 현황 트래킹)를 비개발자 언어로 제공합니다."

### 섹션 7: 최종 CTA

**목적**: 마지막 행동 유도

- **헤딩**: "10크레딧 무료. 카드 등록 없이 바로 시작하세요."
- **서브 텍스트**: "설치 3분. 첫 분석 결과까지 5분."
- **CTA 버튼**: "무료로 시작하기"

## 주의사항

- 랜딩 페이지는 인증 없이 접근 가능해야 함
- 기존 미들웨어에서 `/`을 인증 필수로 처리하고 있으므로, 랜딩 페이지 라우트를 publicPath에 추가해야 함
- "주요 기능" 드롭다운의 하위 페이지(/features/*)는 Phase 2에서 구현. 현재는 드롭다운만 표시하되 링크는 메인 랜딩의 해당 섹션 앵커로 연결
- CTA "무료로 시작하기" → `/auth/login` (Google OAuth)
