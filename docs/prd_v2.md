
# **Asktree**

Claude Code 프로젝트의 코드 파손 감지 + 복구 + 보호 도구

**Product Requirements Document v8.0**

2026.03.29

*"네가 묻지 않아도, AI가 망가뜨린 걸 자동으로 찾아내고, 고치는 법을 알려주고, 다시는 안 망가지게 지켜준다."*

---

## **1. 제품 개요**

### **1.1 한 줄 정의**

**Claude Code로 코딩할 때, AI가 기존 기능을 망가뜨리면 자동으로 감지하고 복구 명령어를 주고, CLAUDE.md에 보호 규칙을 추가해서 재발을 방지하는 도구.**

### **1.2 와우포인트**

**"감지 → Fix → 보호. 한 번 당한 문제는 두 번 없다."**

### **1.3 3단계 가치 체인**

| **단계** | **Asktree가 하는 일** | **유저 행동** | **해소하는 격차** |
| --- | --- | --- | --- |
| **1. 자동 감지** | 세션 간 코드 비교로 삭제/변경된 기능을 자동 발견 | Asktree를 열면 이미 결과가 있음. 아무것도 안 해도 됨. | 인지 격차 (뭐가 망가졌는지 모름) |
| **2. Fix 제공** | 복구를 위한 자연어 CLI 명령어 자동 생성 | Fix를 복사해서 Claude Code에 붙여넣기 | 해결책 격차 (뭘 해야 하는지 모름) |
| **3. CLAUDE.md 보호** | 보호해야 할 기능을 CLAUDE.md 규칙으로 자동 생성 | '보호 추가' 텍스트를 복사해서 CLAUDE.md에 붙여넣기 | 재발 방지 (같은 문제 반복) |

### **1.4 핵심 가치**

| **가치** | **설명** |
| --- | --- |
| 자동 감지 | 유저가 묻지 않아도 코드 파손을 먼저 발견. LLM에게 '뭘 물어봐야 하는지조차 모르는' 문제를 해결. |
| Fix 가이드 | 코드가 아닌 자연어 CLI 명령어. 비개발자가 Claude Code에 그대로 복사-붙여넣기. |
| CLAUDE.md 보호 | 감지된 핵심 기능을 CLAUDE.md 보호 규칙으로 자동 생성. AI가 함부로 삭제 못 하게 방지. |
| 제로 마찰 | 코딩하면 자동 수집·분석. 유저는 기록/정리/관리 불필요. |
| Privacy-First | 코드 원본은 분석 후 즉시 파기(Ephemeral Processing). |

### **1.5 정체성 원칙**

- Asktree는 '보이지 않는 동료'. 바이브코더가 인지조차 못하는 코드 파손을 대신 찾아준다.
- 핵심은 '감지 → Fix → 보호'의 3단계 루프. 시각화나 학습은 목적이 아님.
- Claude Code 전용 (MVP). Cursor/Aider는 Phase 2 확장.
- LLM과의 차별점: LLM은 '물어봐야 답함'. Asktree는 '묻지 않아도 먼저 알려줌' + 세션 간 변경 추적.

## **2. 문제 정의**

### **2.1 리서치 기반 핵심 문제 (Reddit/커뮤니티/인터뷰)**

| **문제** | **발생 빈도** | **구체적 현상** |
| --- | --- | --- |
| **컨텍스트 유실 → 기존 기능 삭제** | 68% | '대시보드 리팩토링해줘' → 3일간 만든 권한 체크+데이터 로직 전부 삭제. 85줄→12줄. |
| **한 파일 수정 → 연쇄 파괴** | 20% | 파일명 변경 → 15개 파일 import 깨짐. DB 컬럼 추가 → 8개 쿼리 에러. |
| AI가 규칙 무시 | 빈번 | CLAUDE.md에 '이 파일 건드리지 마' 써도 긴 대화 후 무시. |
| 문제 인지 자체 불가 | 80% | 코드를 읽을 수 없어서 뭐가 삭제/변경됐는지 모름. |

### **2.2 타겟 유저의 실제 행동 패턴**

- 설계부터 구현까지 AI에게 맡기고, 코드를 읽지 않고 결과(화면) 중심으로만 확인
- 에러가 나면 로그를 복붙해서 AI에게 '고쳐줘'만 반복 → AI death loop
- 컨텍스트 관리를 안 해서 AI가 이전 결정을 잊고 기존 기능을 덮어쓰거나 삭제
- CLAUDE.md의 존재를 모르거나, 알아도 '뭘 써야 할지 몰라서' 사용 못 함 (80%)
- 3~4주 이상 진행하면 AI가 모순된 패치를 내놓아 전체 안정성이 무너짐

### **2.3 왜 LLM만으로는 해결이 안 되는가**

- LLM에게 물어보려면 '뭘 물어봐야 하는지' 알아야 함. 바이브코더 80%는 문제 인지 자체가 불가.
- LLM은 '현재 상태'만 봄. '3일 전에 있던 함수가 오늘 사라졌다'는 추적 불가.
- Asktree는 '묻지 않아도 먼저 알려줌' + '세션 간 변경 추적'. 이 두 가지가 LLM과의 결정적 차이.

## **3. 타겟 유저**

**사이드프로젝트·1인 SaaS 창업형 바이브코더 (Claude Code 사용자)**

- 직군: 기획자, 디자이너, 크리에이터, 직장인 N잡러
- 목적: 유료 서비스, 니치 SaaS 런칭. 실제 사용자에게 돈을 받는 서비스.
- 특성: 개발은 모르지만 비즈니스적 생존을 위해 코딩을 도구로 선택.
- 타겟 프로젝트 규모: 파일 50개 이하, 기능 10개 이하.
- 1차 타겟: 멘토링 다음 기수 수강생 (실제 유통 채널 확보).

## **4. 솔루션**

### **4.1 [이슈] — 코드 파손 감지 + Fix (핵심 와우포인트)**

사이드바에서 [이슈] 선택 시 표시. 세로 이슈 리스트 + Fact→Detail→Fix 아코디언.

**A. 이상 감지 (세션 간 변경 추적):**

- 세션 #N과 세션 #N+1의 코드를 비교하여 삭제/변경된 기능 자동 발견
- 감지 유형: 기존 기능 삭제(68%), 연쇄 파괴(20%), 기술 스택 충돌, PRD 불일치
- Fact(비즈니스 언어) → Detail(기술 근거+세션 맥락+라인 범위) → Fix(자연어 CLI 명령어+복사 버튼)
- Fix 박스에 '복구 명령어 — Claude Code에 붙여넣으세요' 레이블 표시
- 이슈 생명주기: 미확인 → 확인 완료 → 해결됨
- 재감지: Fix 시도 후에도 미해결 시 '이전 Fix와 다른 새로운 명령어' 제공
- 확인 완료 시 CLAUDE.md 보호 규칙 추가 제안

**B. 코드 유의사항 (정적 분석):**

- 세션 비교 없이 현재 코드만으로 감지. 첫 분석부터 즉시 결과 생성 → 첫 사용 가치 보장.
- 카테고리: [보안] API 키 노출, 인증 부재 / [안정성] 에러 처리 없음 / [품질] 미호출 함수
- OWASP Top 10 기준 보안 점검 포함
- 감지 기준이 기술 근거 카드에 명시됨 (ex: 'OWASP Top 10 — 민감 데이터 노출')

**C. CLAUDE.md 보호 규칙:**

- 이슈에서 '확인 완료' 후 '이 기능을 보호 목록에 추가할까요?' 제안
- CLAUDE.md 탭에서 보호 규칙 텍스트 확인 + 복사 버튼 제공
- 가이드라인 본문은 코드 블록 스타일(다크 배경 + 모노스페이스)로 표시
- 전체 미리보기: 터미널 스타일 모달(macOS 윈도우 dot, 다크 배경, 파일명 주황 하이라이트)
- 상태 구분: 미적용 / 적용 완료
- MVP: 유저가 텍스트를 복사해서 CLAUDE.md에 직접 붙여넣기. Phase 2: 자동 추가.
- 프로젝트가 진행될수록 보호 규칙이 쌓여 AI가 함부로 건드리지 못하는 선순환

### **4.2 [현황] — PRD 감리 + 진행률 + 다음 행동 유도**

사이드바에서 [현황] 선택 시 표시. 프로그레스 바 + 카드 그리드 + 클릭 시 상세 펼침.

- 기획서(PRD/FRD) 기능 목록 vs 실제 코드 자동 대조
- 각 기능: 구현됨/부분구현/미구현/확인필요 상태 표시
- 구현 항목별 관련 파일명 + 라인 넘버 인라인 표시 (같은 파일이면 상단 1회 표시 + 항목별 라인 넘버만)
- 메트릭 카드 3개: 구현 항목(3/3), 감지된 이슈(0건), 최근 변경(세션 #4)
- 기획서 참고: FRD 원문 발췌 + '원문 보기 →' 링크
- 기술 상세: 사용 기술, 관련 파일, 관련 세션 가로 3열 카드
- 하단 CTA 배너: 각 상태별 다음 행동 유도
  - 구현 완료: "이 기능은 안전하게 구현되어 있습니다."
  - 부분 구현: "미구현 항목 N개가 남아있습니다." + Claude Code 지시문 안내
  - 확인 필요: "기획서와 코드 간 불일치가 발견되었습니다." + '→ 이슈에서 Fix 확인' 링크
  - 미구현: "아직 코딩이 시작되지 않은 기능입니다." + Claude Code 지시문 안내
- 불일치 발견 시: Fact/Detail/Fix 아코디언 (이슈와 동일 구조)
- 기획서 없이 중간 합류: Reverse IA — 코드+CLI 로그에서 기능 자동 추출
- 기획서에 없는 기능 추가 감지: 'AI가 자체적으로 추가한 것으로 보입니다' 표시

### **4.3 전체 레이아웃**

**Manyfast 스타일 좌측 사이드바 네비게이션 + 우측 메인 콘텐츠.**

- 사이드바: 프로젝트 셀렉터(드롭다운 전환), [현황]/[이슈]/[CLAUDE.md]/[Activity] 메뉴, 기획서, 설정
- [이슈] 메뉴에 미확인 이슈 배지 표시
- [현황]과 [이슈]는 사이드바에서 전환하는 별도 화면. 각 화면이 메인 콘텐츠 전체 사용.
- 기획서 관리: 좌측 첨부 문서 리스트 + 우측 통합 기능 목록(출처 태그 표시). AI 추출 기능 목록을 유저가 편집 가능.
- 글로벌 헤더: #F1F5F9 배경 + 하단 border 구분선 (모든 탭 공통)

### **4.4 사용 흐름**

| **단계** | **사용자 행동** | **Asktree 동작** |
| --- | --- | --- |
| ① 회원가입 | asktree.io, Google 로그인 | 계정 생성 |
| ② 프로젝트 등록 | 프로젝트 생성 + 기획서 업로드 (강력 권장) | 기능 목록 자동 추출 |
| ③ 에이전트 설치 | setup-asktree.sh 원클릭 실행 | 백그라운드 서비스 등록. 기존 CLI 로그 초기 동기화. |
| ④ 코딩 | Claude Code로 평소처럼 작업 | 백그라운드에서 CLI 로그 + 코드 자동 수집. 세션 종료 시 전송. |
| ⑤ 이슈 확인 | [이슈]에서 감지 확인 → Fix 복사 → CLI 붙여넣기 → 보호 규칙 추가 | 감지→Fix→CLAUDE.md 보호 규칙 생성 |
| ⑥ 현황 확인 | [현황]에서 기획 대비 진행률 확인 + 다음 행동 확인 | PRD vs 코드 대조 결과 + CTA 표시 |

### **4.5 입력 채널**

**수동 입력 없음. 자동 수집만.**

| **채널** | **수집 방식** | **MVP** |
| --- | --- | --- |
| 프로젝트 파일 구조 | 로컬 에이전트가 자동 스캔 | ✅ |
| CLI 로그 (Claude Code JSONL) | 로컬 에이전트가 ~/.claude/projects/ 자동 감시 | ✅ |
| GitHub 연동 | Phase 2 보조 채널 | Phase 2 |

### **4.6 에러 유형별 감지 범위**

| **에러 유형** | **감지** | **대응** |
| --- | --- | --- |
| 기존 기능 삭제 (68%) | ✅ | 이상 감지 + Fix + CLAUDE.md 보호 |
| 연쇄 파괴 (20%) | ✅ | 이상 감지 + Fix |
| PRD와 코드 불일치 | ✅ | PRD 감리 ([현황]) |
| 보안 취약 패턴 (OWASP) | ⚠️ 정적만 | 코드 유의사항 |
| 미호출 함수 | ✅ | 코드 유의사항 |
| 실행 순서/런타임 버그 | ❌ 범위 밖 | Sentry 등 모니터링 영역 |

## **5. 기술 아키텍처**

### **5.1 설계 원칙**

| **원칙** | **설명** |
| --- | --- |
| Claude Code 전용 (MVP) | JSONL 로그 + CLAUDE.md 보호. Cursor/Aider는 Phase 2. |
| 로컬 에이전트 + 웹 SaaS | 쉘 스크립트 기반 백그라운드 서비스 + asktree.io 대시보드 |
| Smart Push (단방향) | 로컬→서버 HTTP POST. 세션 종료 감지 시 자동 전송. |

| Ephemeral Processing | 코드 원본 분석 후 즉시 파기. 분석 결과만 DB 저장. |

- 코드/터미널 관련: --gray-900/950 다크 배경 + 모노스페이스
- 초록색 사용 제한: 온보딩/스테퍼에서 사용 안 함. 체크 아이콘은 --foreground(네이비)
- 버튼: Primary는 테두리 없음, '확인 완료'는 Outline 스타일

## **9. 경쟁 분석**

| **서비스** | **Asktree와의 차이** |
| --- | --- |
| CodeRabbit | 개발자 팀용 PR 기반 코드 리뷰. 비개발자 번역 없음. 세션 간 변경 추적 불가. |
| Kodus | 오픈소스 코드 리뷰. 팀 코드베이스 학습. 비개발자 대상 아님. |
| claude-replay | 날것의 로그 리플레이. 감지/Fix 없음. '보여주기만' 함. |
| SonarQube | 개발자용 코드 품질 분석. 비개발자 번역 없음. |
| LLM 직접 질문 | '물어봐야 답함'. Asktree는 '묻지 않아도 먼저 알려줌' + 세션 간 추적. |
| CLAUDE.md 수동 관리 | 바이브코더 80%가 '뭘 써야 할지 몰라' 사용 못 함. Asktree가 자동 생성. |

**Asktree만의 진짜 무기 3가지:**

1. 기획서 대비 코드 대조 (CodeRabbit이 못 하는 것)
2. 세션 간 변경 추적으로 'AI가 망가뜨린 것' 감지 (CodeRabbit이 못 하는 것)
3. 비개발자가 이해할 수 있는 자연어 Fix + CLAUDE.md 보호 (CodeRabbit이 안 하는 것)

## **10. 프롬프트 프로토타이핑 검증 결과**

Kamothon(계약서 위험 탐지기, 10개 파일)으로 Claude Sonnet API 테스트 완료.

- 코드 유의사항 감지: 양호. 6개 실제 문제 감지 (API 키 노출, 미사용 임포트, 빈 모듈 등)
- PRD 감리: 매우 우수. 'RAG 시스템이 정의만 있고 호출 안 됨' 정확히 감지.
- Fix 품질 개선 필요: 코드 블록이 아닌 자연어 CLI 명령문으로 프롬프트 튜닝 필요.
