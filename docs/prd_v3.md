# **Asktree**

Claude Code 프로젝트의 코드 파손 감지 + 복구 + 보호 도구

**Product Requirements Document v9.0**

2026.04.03

_"네가 묻지 않아도, AI가 망가뜨린 걸 자동으로 찾아내고, 고치는 법을 알려주고, 다시는 안 망가지게 지켜준다."_

---

## **1. 제품 개요**

### **1.1 한 줄 정의**

**Claude Code로 코딩할 때, AI가 기존 기능을 망가뜨리면 자동으로 감지하고 복구 명령어를 주고, CLAUDE.md에 보호 규칙을 추가해서 재발을 방지하는 도구.**

### **1.2 와우포인트**

**"감지 → Fix → 보호. 한 번 당한 문제는 두 번 없다."**

### **1.3 3단계 가치 체인**

| **단계**              | **Asktree가 하는 일**                            | **유저 행동**                                      | **해소하는 격차** |
| --------------------- | ------------------------------------------------ | -------------------------------------------------- | ----------------- |
| **1. 자동 감지**      | 세션 간 코드 비교로 삭제/변경된 기능을 자동 발견 | Asktree를 열면 이미 결과가 있음                    | 인지 격차         |
| **2. Fix 제공**       | 복구를 위한 자연어 CLI 명령어 자동 생성          | Fix를 복사해서 Claude Code에 붙여넣기              | 해결책 격차       |
| **3. CLAUDE.md 보호** | 보호해야 할 기능을 CLAUDE.md 규칙으로 자동 생성  | '보호 추가' 텍스트를 복사해서 CLAUDE.md에 붙여넣기 | 재발 방지         |

### **1.4 핵심 가치**

| **가치**       | **설명**                                                                                                                                     |
| -------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| 자동 감지      | 유저가 묻지 않아도 코드 파손을 먼저 발견                                                                                                     |
| Fix 가이드     | 코드가 아닌 자연어 CLI 명령어. 비개발자가 복사-붙여넣기                                                                                      |
| CLAUDE.md 보호 | 감지된 핵심 기능을 CLAUDE.md 보호 규칙으로 자동 생성. 세션 시작 시뿐 아니라 compaction/캐시 무효화 시점에도 다시 로드되어 보호 효과가 강력함 |
| 제로 마찰      | 코딩하면 자동 수집·분석. 유저는 기록/정리/관리 불필요                                                                                        |
| Privacy-First  | 코드 원본은 분석 후 즉시 파기(Ephemeral Processing)                                                                                          |

### **1.5 정체성 원칙**

- Asktree는 '보이지 않는 동료'. 바이브코더가 인지조차 못하는 코드 파손을 대신 찾아준다.
- 핵심은 '감지 → Fix → 보호'의 3단계 루프. 시각화나 학습은 목적이 아님.
- Claude Code 전용 (MVP). Cursor/Aider는 Phase 2 확장.
- LLM과의 차별점: LLM은 '물어봐야 답함'. Asktree는 '묻지 않아도 먼저 알려줌' + 세션 간 변경 추적.

## **2. 문제 정의**

### **2.1 리서치 기반 핵심 문제**

| **문제**                           | **발생 빈도** | **구체적 현상**                                                                  |
| ---------------------------------- | ------------- | -------------------------------------------------------------------------------- |
| **컨텍스트 유실 → 기존 기능 삭제** | 68%           | '대시보드 리팩토링해줘' → 3일간 만든 권한 체크+데이터 로직 전부 삭제. 85줄→12줄. |
| **한 파일 수정 → 연쇄 파괴**       | 20%           | 파일명 변경 → 15개 파일 import 깨짐. DB 컬럼 추가 → 8개 쿼리 에러.               |
| AI가 규칙 무시                     | 빈번          | CLAUDE.md에 '이 파일 건드리지 마' 써도 긴 대화 후 무시.                          |
| 문제 인지 자체 불가                | 80%           | 코드를 읽을 수 없어서 뭐가 삭제/변경됐는지 모름.                                 |

### **2.2 타겟 유저의 실제 행동 패턴**

- 설계부터 구현까지 AI에게 맡기고, 코드를 읽지 않고 결과(화면) 중심으로만 확인
- 에러가 나면 로그를 복붙해서 AI에게 '고쳐줘'만 반복 → AI death loop
- 컨텍스트 관리를 안 해서 AI가 이전 결정을 잊고 기존 기능을 덮어쓰거나 삭제
- CLAUDE.md의 존재를 모르거나, 알아도 '뭘 써야 할지 몰라서' 사용 못 함 (80%)

### **2.3 왜 LLM만으로는 해결이 안 되는가**

- LLM에게 물어보려면 '뭘 물어봐야 하는지' 알아야 함. 바이브코더 80%는 문제 인지 자체가 불가.
- LLM은 '현재 상태'만 봄. '3일 전에 있던 함수가 오늘 사라졌다'는 추적 불가.

## **3. 타겟 유저**

**사이드프로젝트·1인 SaaS 창업형 바이브코더 (Claude Code 사용자)**

- 직군: 기획자, 디자이너, 크리에이터, 직장인 N잡러
- 목적: 유료 서비스, 니치 SaaS 런칭. 실제 사용자에게 돈을 받는 서비스.
- 특성: 개발은 모르지만 비즈니스적 생존을 위해 코딩을 도구로 선택.
- 타겟 프로젝트 규모: 파일 50개 이하, 기능 10개 이하.

## **4. 솔루션**

### **4.1 [이슈] — 코드 파손 감지 + Fix (핵심 와우포인트)**

사이드바에서 [이슈] 선택 시 표시. 좌측 3섹션 이슈 리스트 + 우측 Fact→Detail→Fix 상세 패널.

**좌측 리스트 구조 (모든 뷰 공통):**

- 3섹션: "미확인 (N)" / "확인 완료 (N)" / "해결됨 (N)"
- 각 섹션 접기/펼치기 가능 (chevron-down/right)
- 섹션 헤더: #E7E5E4 배경, 12px Bold #78716C
- 선택 항목: #FFFFFF 배경 + 좌측 이슈 레벨 컬러 3px 세로선
- 확인 완료 항목: 제목 #78716C, 체크 아이콘
- 해결됨 항목: 제목 #A8A29E + 취소선

**우측 상세 패널 구조:**

- Fact(비즈니스 언어) → Detail(기술 근거) → Fix(자연어 CLI 명령어)
- Fact 카드: 좌측 이슈 레벨 컬러 3px 보더 (Critical=#DC2626, Warning=#F97316)
- Fix 박스: --primary 투명도 5% 배경 + 모노스페이스 폰트 + "복구 명령어 — Claude Code에 붙여넣으세요" 레이블 + 복사 버튼
- 기술 근거 카드: 고정 3열 "관련 파일 / 감지 근거 / 감지 시간"
- 안내 문구: ⓘ + "확인 시 이 기능의 보호 규칙(CLAUDE.md) 추가를 제안합니다."
- "확인 완료" 버튼: 패널 맨 하단, #44403C 배경 흰색 텍스트

**이슈 생명주기:** 미확인 → 확인 완료 → 해결됨 / 미해결 시 재감지(미확인 복귀 + "재감지" 배지)

**코드 유의사항 (정적 분석):**

- 세션 비교 없이 현재 코드만으로 감지. 첫 분석부터 즉시 결과.
- 카테고리: [보안] API 키 노출, 인증 부재 / [안정성] 에러 처리 없음 / [품질] 미호출 함수
- OWASP Top 10 기준 보안 점검 포함

**CLAUDE.md 보호 규칙:**

- "확인 완료" 후 보호 규칙 추가 제안 모달 표시
- CLAUDE.md 탭에서 코드 블록 스타일(다크 배경+모노스페이스)로 보호 규칙 확인 + 복사
- 전체 미리보기: 터미널 스타일 모달 (macOS dot, 다크 배경, 파일명 주황 하이라이트)
- 상태 구분: 미적용 / 적용 완료
- MVP: 유저가 텍스트를 복사해서 CLAUDE.md에 직접 붙여넣기

### **4.2 [현황] — PRD 감리 + 진행률 + 다음 행동 유도**

사이드바에서 [현황] 선택 시 표시. 좌측 기능 리스트 + 우측 상세 패널.

- 기획서(PRD/FRD) 기능 목록 vs 실제 코드 자동 대조
- 각 기능: 구현됨/부분구현/미구현/확인필요 상태 표시
- 타이틀에 상태 컬러 적용 + ⓘ info 아이콘(hover 시 툴팁)
  - 구현 완료: 네이비 블루 + "다음 코딩 세션 후 자동으로 재분석됩니다"
  - 부분 구현: 주황 + "미구현 항목은 Claude Code에 지시하세요"
  - 확인 필요: 빨강 + "이슈 탭에서 상세 내용을 확인하세요"
  - 미구현: 회색 + "코딩을 시작하면 자동으로 감지됩니다"
- 구현 항목별 관련 파일명 + 라인 넘버 인라인 표시
- 메트릭 카드 3개, 기획서 참고 카드, 기술 상세 카드
- 기획서 없이 중간 합류: Reverse IA — 코드+CLI 로그에서 기능 자동 추출
- 기획서X 코드O 화면: "기획서 업로드하기 →" 링크 포함

### **4.3 전체 레이아웃**

**Manyfast 스타일 좌측 사이드바 네비게이션 + 우측 메인 콘텐츠.**

- 사이드바 메뉴: [현황] / [이슈] / [CLAUDE.md] / [세션] / 기획서 / 설정
- [이슈] 메뉴에 미확인 이슈 배지, [CLAUDE.md]에 미적용 가이드라인 배지
- 프로젝트 셀렉터: chevron-down(▾) 아이콘 포함, 클릭 시 프로젝트 전환 드롭다운
- 프로젝트 드롭다운 내용: "← 모든 프로젝트" + 최근 프로젝트 리스트 + "+ 새 프로젝트"
- 사이드바 하단: "에이전트 연결됨/미연결" + "크레딧: N" (클릭 시 관리자 설정 이동)
- 글로벌 헤더: #F5F5F4 배경 + 하단 #E7E5E4 border (모든 탭 공통)
- 기획서 관리: 좌측 첨부 문서 리스트 + 우측 통합 기능 목록(출처 태그 FRD/PRD pill)

### **4.4 사용 흐름**

| **단계**        | **사용자 행동**                                                 | **Asktree 동작**                         |
| --------------- | --------------------------------------------------------------- | ---------------------------------------- |
| ① 회원가입      | asktree.io, Google 로그인                                       | 계정 생성                                |
| ② 프로젝트 등록 | 프로젝트 생성 + 기획서 업로드 (강력 권장)                       | 기능 목록 자동 추출                      |
| ③ 에이전트 설치 | setup-asktree.sh 원클릭 실행                                    | 백그라운드 서비스 등록                   |
| ④ 코딩          | Claude Code로 평소처럼 작업                                     | 백그라운드에서 CLI 로그 + 코드 자동 수집 |
| ⑤ 이슈 확인     | [이슈]에서 감지 확인 → Fix 복사 → CLI 붙여넣기 → 보호 규칙 추가 | 감지→Fix→CLAUDE.md 보호 규칙 생성        |
| ⑥ 현황 확인     | [현황]에서 기획 대비 진행률 확인                                | PRD vs 코드 대조 결과 표시               |

### **4.5 입력 채널**

**수동 입력 없음. 자동 수집만.**

| **채널**                     | **수집 방식**                                 | **MVP** |
| ---------------------------- | --------------------------------------------- | ------- |
| 프로젝트 파일 구조           | 로컬 에이전트가 자동 스캔                     | ✅      |
| CLI 로그 (Claude Code JSONL) | 로컬 에이전트가 ~/.claude/projects/ 자동 감시 | ✅      |
| GitHub 연동                  | Phase 2 보조 채널                             | Phase 2 |

### **4.6 로컬 에이전트 데이터 전송 범위**

| **데이터**                            | **전송 여부** | **비고**                                                                  |
| ------------------------------------- | ------------- | ------------------------------------------------------------------------- |
| JSONL 로그 (Claude Code 대화 기록)    | ✅ 전송       | 세션 종료 감지 시 자동 전송                                               |
| 프로젝트 파일 구조 (파일명/경로/크기) | ✅ 전송       | 파일 내용(코드 원본)은 포함하지 않음                                      |
| 변경된 파일의 diff                    | ✅ 전송       | diff만으로 분석 가능한 경우 diff만 전송                                   |
| 코드 원본 전체                        | 조건부 전송   | Ephemeral Processing 대상. 분석 후 즉시 파기. diff만으로 부족할 때만 전송 |

> **원칙:** 코드 원본이 서버에 전송되는 경우, 분석 완료 즉시 파기(Ephemeral Processing). 분석 결과(메타데이터)만 DB에 저장.

## **5. 기술 아키텍처**

### **5.1 설계 원칙**

| **원칙**                | **설명**                                                                          |
| ----------------------- | --------------------------------------------------------------------------------- |
| Claude Code 전용 (MVP)  | JSONL 로그 + CLAUDE.md 보호. Cursor/Aider는 Phase 2.                              |
| 로컬 에이전트 + 웹 SaaS | 쉘 스크립트 기반 백그라운드 서비스 + asktree.io 대시보드                          |
| Smart Push (단방향)     | 로컬→서버 HTTP POST. 세션 종료 감지 시 자동 전송.                                 |
| IDE 독립적 구조         | 백그라운드 서비스. IDE/터미널 종료와 무관.                                        |
| Ephemeral Processing    | 코드 원본 분석 후 즉시 파기. 분석 결과만 DB 저장.                                 |
| API 키 암호화 저장      | Supabase pgcrypto 확장으로 AES-256 서버 사이드 암호화. 복호화 키는 환경변수 관리. |

| 웹 프레임워크 | Next.js               |
| 서버 DB       | Supabase (PostgreSQL) |
| 파일 감시     | chokidar              |

- 좌측 리스트 선택 항목: #FFFFFF 배경 + 좌측 컬러 3px 세로선
- 빈 상태: Master-Detail 유지 (좌측 빈 섹션 + 우측 안내 메시지). CTA 버튼 최대 240px.

### **6.6 인터랙션 스펙 (개발 시 적용)**

- Primary 버튼: hover #C2410C, active #9A3412
- Outline 버튼: hover 배경 #F5F5F4, active #E7E5E4
- Destructive 버튼: hover #B91C1C
- 텍스트 링크(#E67D22): hover 시 밑줄
- 좌측 리스트 항목: hover 배경 #FAFAF9
- 카드: hover shadow 강화 (0 2px 8px rgba(0,0,0,0.1))

## **8. 비즈니스 모델**

**맛보기 크레딧 10회 → 본인 API 키 연동 또는 월정액 구독**

- 크레딧 단위: Claude API 토큰 소비량의 환산 단위. 프로젝트 규모(파일 수, 세션 수)에 따라 1회 분석의 소모량이 다름.
- 분석 실행 확인 모달에 "예상 소요 크레딧: 약 N 크레딧"을 표시하여 유저가 소모량을 사전 인지.
- 1차 타겟: 멘토링 다음 기수 수강생 (유통 채널 확보)
- 핵심 검증 지표: '2주 이상 반복 사용하는 유저 비율'

## **9. 경쟁 분석**

| **서비스**          | **Asktree와의 차이**                                                       |
| ------------------- | -------------------------------------------------------------------------- |
| CodeRabbit          | 개발자 팀용 PR 기반 코드 리뷰. 비개발자 번역 없음. 세션 간 변경 추적 불가. |
| Kodus               | 오픈소스 코드 리뷰. 팀 코드베이스 학습. 비개발자 대상 아님.                |
| LLM 직접 질문       | '물어봐야 답함'. Asktree는 '묻지 않아도 먼저 알려줌' + 세션 간 추적.       |
| CLAUDE.md 수동 관리 | 바이브코더 80%가 '뭘 써야 할지 몰라' 사용 못 함. Asktree가 자동 생성.      |

**Asktree만의 진짜 무기 3가지:**

1. 기획서 대비 코드 대조 (CodeRabbit이 못 하는 것)
2. 세션 간 변경 추적으로 'AI가 망가뜨린 것' 감지 (CodeRabbit이 못 하는 것)
3. 비개발자가 이해할 수 있는 자연어 Fix + CLAUDE.md 보호 (CodeRabbit이 안 하는 것)
